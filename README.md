## Explain the project

This project has two main features ( Sign up & Sign in ) , navigating through links by routes. There is a initial username and password that can Sign in with them, it will navigate to Dashboard page
#### Additional features
- User can sign up by his username, email and password
- User can sign in using his username and password
- If he forgot passwor, he can qrite his registerd email to recreate a new password, so he can sign in again.
- if he type anything wrong in url path, A NotFoundPage will be displayed and redirect to login page

## Some Modifications
the NotFoundPage wasn't displayed and go to  404 error on Vercel despite having proper client-side routing configuration when accessing undefined routes is happening because while your React app's client-side routing is set up correctly, As a result, we create vercel.json 
```js
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "no-cache"
        }
      ]
    }
  ]
}

```
Add ```<base href="/" />``` to ```<head>``` in **index.html**

```js
  {
    files: ['**/*.{ts,tsx}'],
    extends: [

      // Alternatively, use this for stricter rules
      ...tseslint.configs.strictTypeChecked,
      // Optionally, add this for stylistic rules
      ...tseslint.configs.stylisticTypeChecked,

      // Other configs...
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```

You can also install [eslint-plugin-react-x](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-x) and [eslint-plugin-react-dom](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-dom) for React-specific lint rules:

```js
// eslint.config.js
import reactX from 'eslint-plugin-react-x'
import reactDom from 'eslint-plugin-react-dom'

export default tseslint.config([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      // Other configs...
      // Enable lint rules for React
      reactX.configs['recommended-typescript'],
      // Enable lint rules for React DOM
      reactDom.configs.recommended,
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```
